# Tech Stack: Org-X

**Last Updated:** 2025-03-22

## Overview

Org-X is a cross-platform desktop application for viewing Org-mode files. It uses **Tauri 2.0** (Rust backend) with **Svelte 5** (frontend) architecture.

## Backend Stack

### Core Runtime
- **Language:** Rust (Edition 2021)
- **Framework:** Tauri 2.0
- **Process Model:** Multi-threaded Tokio runtime

### Key Dependencies

| Library | Version | Purpose |
|---------|---------|---------|
| `tauri` | 2.0 | Application framework |
| `tokio` | 1.35.1 | Async runtime |
| `orgize` | 0.9.0 | Org-mode parser |
| `serde` / `serde_json` | 1.0 | Serialization |
| `notify` | 6.1.1 | File system monitoring |
| `thiserror` | 2.0.12 | Error handling |
| `uuid` | 1.16.0 | Unique identifiers |
| `chrono` | 0.4.40 | Date/time handling |
| `specta` | 2.0.0-rc.22 | Type generation |
| `tauri-specta` | 2.0.0-rc.21 | TS bindings generation |

### Type Safety Bridge
- **tauri-specta:** Auto-generates TypeScript bindings from Rust types
- Output: `src/lib/bindings.ts` (684 lines, auto-generated)
- Updates on every dev build in debug mode

## Notable Patterns

### Auto-Generated Bindings
The `src/lib/bindings.ts` file is auto-generated by tauri-specta. **Never edit manually.** It exports:
- `commands` object with all Tauri invoke handlers
- Type definitions for all Rust structs
- Result<T, E> type for error handling

### Frontend-Backend Communication
- All communication through Tauri commands
- Type-safe via generated bindings
- Commands return `Result<T, string>` for error handling
- File: `src-tauri/src/api.rs` defines all commands
